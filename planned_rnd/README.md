# Будущие RnD (ещё не реализованы)

Эта папка зарезервирована под новые RnD-направления.

## Принцип именования
- `rnd_02_<название>`
- `rnd_03_<название>`
- и т.д.

## Минимальный состав каждого нового RnD
1. `README.md` — навигация по RnD (главная точка входа).
2. `report.md` — краткий отчёт с выводами (вторая точка входа).
3. `analysis.ipynb` — воспроизводимый ноутбук с актуальными output.
4. (Опционально) скрипты симуляций и артефакты (`*.csv`, `*.pdf`).
5. Ссылки из отчёта на соответствующий код.

## Рекомендуемый шаблон структуры

```text
rnd_0X_<название>/
  README.md
  report.md
  analysis.ipynb
  (опционально) simulation.py
  (опционально) results.csv / report.pdf
```

Такой шаблон обеспечивает понятный вход: сначала резюме, затем переход к воспроизводимому коду.
