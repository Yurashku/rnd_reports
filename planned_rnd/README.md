# Памятка по созданию новых RnD

Этот документ — мини-tutorial по обязательному формату репозитория.

## Обязательный layout для каждого нового RnD

Каждое новое исследование создаётся отдельной папкой в корне и содержит **ровно 3 файла**:

```text
0X_<topic>/
  notebook.ipynb
  report.md
  report.pdf
```

## Рекомендуемый процесс

1. Создать `report.md` как основной источник содержания.
2. Подготовить `notebook.ipynb` на синтетике с фиксированным seed.
3. Выполнить ноутбук до конца и сохранить output.
4. Сгенерировать `report.pdf` из `report.md` командой:
   ```bash
   python tools/generate_pdf.py
   ```

## Важно
- Внутри RnD-папки не хранить дополнительные файлы (`csv`, `png`, `py`, `txt` и т.п.).
- Графики и вычислительные артефакты должны оставаться внутри ноутбука.
- Служебные вещи размещать только в корне (`tools/`, `docs/`, `.github/`, `scripts/`).
