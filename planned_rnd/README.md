# Будущие RnD (ещё не реализованы)

Папка зарезервирована под следующие исследования.

## Принцип структуры: «один RnD — один основной файл»
Для каждого нового RnD в корне папки обязательно должен быть **единый `report.md`**, который содержит почти всю информацию:
- цель и постановку;
- ключевую математику/методологию;
- выводы и ограничения;
- ссылки только на код и вычислительные артефакты.

## Именование
- `rnd_02_<название>`
- `rnd_03_<название>`
- и далее по порядку.

## Минимальный состав нового RnD
1. `report.md` — главный консолидированный документ.
2. `analysis.ipynb` — воспроизводимый ноутбук (полностью выполненный, с output).
3. `README.md` — короткая памятка (опционально, если нужна отдельная навигация).
4. (Опционально) `simulation.py`, `results.csv`, `report.pdf`.

## Шаблон
```text
rnd_0X_<название>/
  report.md
  analysis.ipynb
  (опционально) README.md
  (опционально) simulation.py
  (опционально) results.csv / report.pdf
```
