# RnD 04: план внедрения методологии в продукт

## Что фиксируем в дизайн-документе
1. Набор primary/secondary метрик.
2. Тип контроля ошибок (FWER/FDR).
3. Правило приемки сплита (например, пороги SMD + лимит попыток).
4. Единый шаблон отчёта эксперимента.

## Минимальный pipeline
1. Проверка баланса ковариат до запуска.
2. Запуск эксперимента по зафиксированному протоколу.
3. Финальный анализ с корректировками для множественных гипотез.
4. Публикация итогового отчёта с ссылками на код и артефакты.
