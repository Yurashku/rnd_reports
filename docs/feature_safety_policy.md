# Политика безопасности in-time ковариат

- **Версия документа:** v0.1
- **Статус:** Draft (заглушка)
- **Связанный код:** `src/rnd_reports/feature_safety/`

## Зачем нужна политика
Ковариата для снижения дисперсии безопасна, только если её использование не
вносит смещение в оценку эффекта. Политика задаёт критерии, по которым признак
относят к безопасным, и описывает обязательные проверки.

## Черновик критериев безопасности
- **Независимость от назначения:** значение ковариаты не зависит от группы (Z/D).
- **Временна́я валидность (in-time):** ковариата фиксируется в допустимый момент и не «подсматривает» будущее, связанное с эффектом.
- **Воспроизводимость:** признак вычислим из доступных полей по детерминированному правилу.
- **Отсутствие утечки целевой метрики:** ковариата не является прокси будущего значения `Y` после воздействия.

## Где это живёт в коде (заглушки)
- `contracts.py` — контракт безопасной ковариаты (`FeatureContract`).
- `registry.py` — реестр безопасных признаков (`FeatureRegistry`).
- `rules.py` — правила проверки (`SAFETY_RULES`, `check`).
- `diagnostics.py` — диагностики (`diagnose`).
