# Памятка по созданию новых RnD

Этот документ — мини-tutorial по формату репозитория. Полный процесс — в
[`../docs/WORKFLOW.md`](../docs/WORKFLOW.md).

## Layout каждого нового RnD

Каждое исследование создаётся отдельной папкой внутри `rnd/`:

```text
rnd/0X_<topic>/
  notebook.ipynb            # обязателен — тонкая витрина поверх src/rnd_reports/
  notebook_internal.ipynb   # опционален — самодостаточный ноутбук для внутреннего контура
  report.md                 # обязателен — источник правды отчёта
  report.pdf                # опционален — собирается только для RnD из PDF_ENABLED
  README.md                 # обязателен — краткое описание
```

## Рекомендуемый процесс

1. Создать `report.md` как основной источник содержания.
2. Подготовить `notebook.ipynb` на синтетике с фиксированным seed.
3. Если RnD «табличный» (его проверяют на одной таблице во внутреннем контуре) — завести
   `notebook_internal.ipynb`: самодостаточный, читает таблицу из `INPUT_CSV`, с отдельной
   ячейкой-генератором синтетической таблицы (см. `docs/WORKFLOW.md`).
4. Выполнить **все** ноутбуки до конца и сохранить output:
   ```bash
   .venv/bin/python tools/execute_notebooks.py
   ```
5. Если нужен PDF — добавить `0X_<topic>` в `PDF_ENABLED` (`tools/generate_pdf.py`) и собрать:
   ```bash
   .venv/bin/python tools/generate_pdf.py
   ```

## Важно
- Внутри RnD-папки не хранить лишние файлы (`csv`, `png`, `py`, `txt`); исключение —
  `notebook_internal.ipynb`.
- Графики и вычислительные артефакты остаются внутри ноутбука; компактные сводки/figures —
  в `results/0X_<topic>/`; переиспользуемый код — в `src/rnd_reports/`.
- Все комментарии в ноутбуках — на русском.
- Перед push все изменённые ноутбуки перезапускаются и проходят до конца.
