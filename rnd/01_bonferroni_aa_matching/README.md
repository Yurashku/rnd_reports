# RnD-1: Поправки на множественные проверки в A/A и A/B, связь с мэтчингом

Как контролировать множественность при A/A-проверке баланса сплита и в итоговом A/B по
нескольким endpoint'ам, и как это связано с балансировкой/мэтчингом. Показано, почему рост
числа ковариат раздувает долю «ложных» отличий и провоцирует лишние рерандомизации.

- **Статус:** ✅ done (draft-отчёт).

- **Код-витрина:** [`notebook.ipynb`](notebook.ipynb) (полный исторический демо-ноутбук — в `archive/`).
